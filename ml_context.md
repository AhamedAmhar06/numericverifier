# ML Decision Model – Context

## Purpose
Train a lightweight supervised ML model that replaces or improves the rule-based
decision logic for NumericVerifier.

The model predicts:
- ACCEPT
- REPAIR
- FLAG

based solely on verifier-generated numeric signals.

---

## Problem Definition

- Task: Multiclass classification
- Input features: verifier signals (numeric, structured)
- Output labels: ACCEPT / REPAIR / FLAG
- Model role: decision-making only (NOT verification)

---

## Dataset

### Primary dataset
- File: signals.csv
- Generated by backend verifier
- Each row = one verification run

### Feature columns (inputs)
The model must use ONLY the following features:

- unsupported_claims_count
- coverage_ratio
- recomputation_fail_count
- max_relative_error
- mean_relative_error
- scale_mismatch_count
- period_mismatch_count
- ambiguity_count

NO raw text, NO evidence, NO answers are allowed as features.

### Label column
- decision (string): ACCEPT | REPAIR | FLAG

---

## Data Leakage Constraints (STRICT)

- Do NOT use raw text, question, evidence, or candidate answers.
- Do NOT derive features from gold labels.
- Do NOT use timestamps or IDs.
- Only verifier signals are allowed.

This model must not “see” the answer content.

---

## Preprocessing

- Drop non-feature columns
- Encode labels:
  - ACCEPT → 0
  - REPAIR → 1
  - FLAG → 2
- Scale features using StandardScaler
- Stratified train/validation split (e.g., 80/20)

---

## Class Imbalance Handling

- Measure class distribution
- Use class_weight="balanced" where applicable
- Do NOT oversample unless imbalance is extreme
- Report per-class metrics

---

## Models to Train

Train and compare at least:

1. Logistic Regression (multinomial, interpretable baseline)
2. Random Forest (non-linear decision boundaries)

Avoid deep learning models.

---

## Evaluation Metrics

Must report:
- Accuracy (secondary)
- Macro F1-score (primary)
- Confusion matrix
- Per-class recall (especially FLAG)

Compare ML model vs rule-based baseline.

---

## Outputs (REQUIRED)

Export:
- decision_model.joblib
- feature_schema.json (ordered feature list)
- evaluation summary (metrics + confusion matrix)

These artifacts will be imported into the backend for inference.

---

## Constraints

- Use Python + scikit-learn only
- Notebook must be fully runnable top-to-bottom
- No GPU-only dependencies
- No backend code changes in this step

---

## Goal

Demonstrate that a learned decision model improves over deterministic rules
while preserving interpretability and avoiding data leakage.
